# 2026-05-19 11:30 (UTC) 页面鉴权高危缺陷修复

## 背景

本次自动化任务要求审查近期提交，重点寻找会导致数据泄露、崩溃、数据丢失或重大用户影响的高危 correctness bug。审查范围聚焦最近登录能力、线上 Postgres 认证存储和首页登出相关提交。

## 发现的问题

- `src/middleware.ts` 使用 Better Auth 的 `getSessionCookie` 只检查 cookie 是否存在，无法校验签名、过期状态或数据库会话。
- 多个受保护 Server Page 在进入页面时直接读取 Volcengine 集群列表或 Kubernetes 资源；伪造或过期的 session cookie 可绕过 middleware，并在 SSR 阶段拿到敏感数据。
- API Route 已通过 `assertAuthenticated()` 做真实会话校验，但 SSR 页面路径没有同等级别保护。

## 修复内容

- 新增 `src/lib/require-page-session.ts`，在页面渲染前调用 `auth.api.getSession()` 做真实会话校验，未登录时重定向 `/login`。
- 在首页、集群详情、资源列表、创建页面、YAML 创建页面和 AI Chat 页面入口调用 `requirePageSession()`。
- 相关受保护页面显式设置 `dynamic = "force-dynamic"`，避免构建期预渲染触发认证或数据读取。
- 新增单测覆盖无效会话下不会调用 Volcengine/K8s 后端读取。
