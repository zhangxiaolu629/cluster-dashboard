# 架构文档生成器

当用户要求为本项目创建或更新架构文档时，使用此 skill。

## 目标

基于真实代码仓库分析结果，在项目根目录产出完整的 `ARCHITECTURE.md`。

## 必要分析范围

1. 分析当前项目目录结构，并提供：
   - 顶层目录说明（每个文件夹的用途）
   - 从 `package.json` 提取技术栈
   - 核心配置文件作用（如 `next.config`、`tsconfig`、`tailwind.config`）
2. 对以下目录生成详细模块说明：
   - `src/app`（路由结构）
   - `src/components`（组件组织方式）
   - `src/lib`（工具函数和第三方配置）
   - `src/server`（如果存在，列出服务端逻辑）
3. 对每个模块都要说明：
   - 主要职责
   - 关键文件及其作用
   - 与其他模块的依赖关系

## 输出格式

将所有分析整合为一个 `ARCHITECTURE.md`，并严格使用以下结构：

```markdown
# 项目架构文档

## 1. 概述

- 项目名称、技术栈、构建工具

## 2. 目录结构

（树形图 + 说明）

## 3. 核心模块

（每个模块的子章节）

## 4. 数据流

- 客户端到服务端的请求路径
- 状态管理方式

## 5. 外部依赖

- 关键第三方库及其用途

## 6. 开发规范

- 从现有代码中总结出的隐含约定
```

## 执行规则

- 所有结论必须基于仓库真实内容，不得臆造文件或行为。
- 若 `src/lib` 或 `src/server` 不存在，需明确说明，并继续完成其余章节。
- 输出全文必须使用中文（包含标题、说明与解释）。
- 表述保持简洁、实用。
- 最终将文档直接保存到项目根目录，文件名为 `ARCHITECTURE.md`。
